AI 환자 문진 시스템: Smart Chart

About Me

저는 사람들이 불편해하는 문제를 기술로 해결하는 과정에서 매력을 느끼는 개발자입니다.

이 프로젝트를 통해 환자들이 자신의 증상을 더욱 잘 전달하고 짧은 진료 시간 안에 떨지 않고 증상을 말할 수 있게 되는 것이 제가 원하는 목표 입니다.

1. 문제 정의 (The Problem)

기존 진료 시스템은 의사에게 반복적인 질문 부담과 번아웃(Burnout)을, 환자에게는 짧은 진료 시간 내에 증상을 설명해야 하는 심리적 압박감을 유발했습니다. 특히 정신과나 산부인과처럼 환자의 '서사(narrative)'가 중요한 진료과에서는, 환자가 긴장하여 핵심 증상을 잊거나 의사와의 정서적 교감이 단절되는 문제가 발생합니다.

2. 프로젝트 목표 (The Goal)

이 프로젝트의 목표는 AI를 활용해 환자의 복잡한 이야기를 '스마트 리포트'로 요약하여, 의사가 진료 시간을 '질문'이 아닌 '진단과 공감'에 집중할 수 있도록 돕는 것입니다.

기술 스택 (Tech Stack)

구분 기술 사용처
프론트엔드 JavaScript, React, HTML, CSS 환자 및 의료진용 UI/UX 구현 (React는 본 프로젝트의 기반 기술)
백엔드 (BaaS) Firebase (Firestore, Authentication) 실시간 대기 명단 DB, 의료진 인증
AI Google Gemini API 환자 문진표 요약 및 '스마트 리포트' 생성
Tools & DevOps Git, VScode, Google Cloud Console 버전 관리, 개발 환경, API 키 관리

아키텍처 (Architecture)

백엔드 서버를 직접 구축하는 대신, Firebase를 BaaS(Backend as a Service)로 채택한 서버리스 아키텍처를 설계했습니다.

시스템 아키텍처 다이어그램

(의료진/환자 Client - Firebase (Auth/Firestore) - Gemini API 간의 데이터 흐름도)

  • 데이터베이스 (Firestore): 실시간 동기화가 필수적인 '대기자 명단' 관리를 위해 Cloud Firestore를 사용했습니다. (`/public/data/waiting_list`)
  • 인증 (Authentication): 의사/데스크 직원의 보안 로그인을 위해 Firebase Authentication (이메일/비밀번호 방식)을 사용했습니다.
  • AI 연동 (Gemini API): 환자 문진표 요약을 위해, Google의 Gemini API를 클라이언트(`doctor.html`)에서 직접 `fetch` API로 호출했습니다.
  • 프론트엔드 (HTML/Tailwind): `index.html`, `doctor.html`, `patient.html` 3개의 정적 파일로 분리하고 Tailwind CSS로 UI를 구현했습니다.

핵심 기능 (Core Features)

1. 실시간 환자 대기 명단 관리

`doctor.html`에서 데스크 직원이 환자를 등록(`addDoc`)하면, 이 정보가 Firestore의 `waiting_list` 컬렉션에 저장됩니다. `patient.html`은 이 컬렉션을 `onSnapshot`으로 실시간 구독(listening)하여, 새 환자가 등록되는 즉시 사이드바에 이름을 띄웁니다. 환자가 문진을 시작(`status: "writing"`)하거나 완료(`status: "completed"`)할 때마다 `updateDoc`으로 상태를 변경하며, 이 상태는 `doctor.html` 대시보드에도 실시간으로 반영됩니다.

2. Gemini API 연동 'AI 스마트 리포트'

`doctor.html`에서 '작성 완료'된 환자를 클릭하면, Firestore에서 해당 환자의 `questionnaireData`를 조회합니다. 이 데이터를 기반으로 '의사 보조' 역할을 하는 **시스템 프롬프트(System Prompt)**를 설계하고, Gemini API에 `fetch`로 요약 요청을 전송합니다. 반환된 AI 요약 텍스트(JSON)를 파싱하여 리포트 모달 최상단에 렌더링, 의사가 환자 진료 전 10초 이내에 핵심을 파악할 수 있도록 했습니다.

나의 역할 및 기여 (My Role)

  • 프로젝트 기획 및 Firebase(BaaS) 기반 서버리스 아키텍처 설계
  • `doctor.html`, `patient.html` 3개 페이지의 UI/UX 흐름 설계 및 Tailwind CSS 구현
  • Firebase Authentication(의사 인증) 및 Firestore(실시간 대기 명단) 스키마 설계 및 연동
  • `onSnapshot`을 이용한 의사-환자 페이지 간 실시간 데이터 동기화 로직 구현
  • 정신과용 '하이브리드 질문지' 7단계 문항 설계
  • Gemini API 연동 및 AI 요약본을 위한 시스템 프롬프트 엔지니어링

회고 (Retrospective)

결과 및 성과 (Results)

  • 서버 없이 Firebase만으로 실시간 풀스택 앱 프로토타입 완성
  • 환자의 주관식 답변을 AI가 '스마트 리포트'로 요약하는 핵심 기능 구현
  • 의사의 정보 파악 시간 단축 및 환자의 자기표현 시간 확보 목표 달성

배운 점 (Learnings)

  • `onSnapshot` 리스너의 강력함과 BaaS의 백엔드 개발 공수 단축 효과 체감
  • 비정형 데이터를 AI가 이해하도록 '프롬프트 엔지니어링'을 적용하는 과정 경험
  • API 키 호출 관련 CORS 및 403 오류 디버깅 (GCP 콘솔 활성화)

개선점 (Future Work)

  • (보안) API 키가 클라이언트에 노출됨. 중간 API 서버(Node.js 등) 구축 필요
  • (기능) AI 요약본을 의사가 수정/보완할 수 있는 '리포트 편집' 기능 추가
